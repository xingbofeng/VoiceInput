# VoiceInput 完整 Bug List

日期：2026-06-12

## 1. 文档目的

本文档汇总用户连续反馈中的全部问题，作为后续修复、验收和回归测试的唯一清单。

状态定义：

- `待修复`：尚未按要求完成。
- `仍不通过`：已有修改，但用户复查后明确反馈仍不正确。
- `待视觉重做`：功能可能存在，但布局、颜色、层级或交互质量不达标。
- `待验证`：代码可能已覆盖，需要在真实 App 中再次确认。

优先级定义：

- `P0`：阻塞使用、数据或核心流程错误。
- `P1`：主要功能或高频交互明显不正确。
- `P2`：视觉质量、布局和低频交互问题。

## 2. 全局验收原则

1. 所有可点击控件必须有足够大的点击区域，不能只允许点击图标线条或文字。
2. 主要按钮、危险按钮、成功状态和普通选中状态必须使用不同语义颜色。
3. 页面应使用清晰的卡片层级、统一圆角、克制阴影和低饱和强调色。
4. 不允许出现点击无反应、保存无反馈、必填项缺失却没有提示等“静默失败”。
5. 输入框必须符合 macOS 常规行为，包括粘贴、`Command+A` 全选和正常替换文本。
6. 弹窗应支持右上角关闭；适合轻量编辑的弹窗应支持点击空白区域关闭。
7. 所有状态和操作文案使用中文，不直接显示内部英文枚举。
8. 用户提供的截图是布局和交互参考，不要求逐像素复制，但信息架构、操作位置和视觉层级必须一致。

## 3. 构建产物与启动

### BUG-BUILD-001 `.build` 打开时显示无权限

- 优先级：`P0`
- 状态：`待验证`
- 问题：打开 `.build` 中的应用或相关产物时显示没有权限。
- 验收标准：
  - `.build/VoiceInputApp.app` 能正常启动。
  - App bundle 内可执行文件具有执行权限。
  - 构建后完成签名验证，不出现 macOS 权限或损坏提示。
  - 明确区分 Finder 目录访问权限、可执行权限、签名或 Gatekeeper 问题。

### BUG-APP-001 应用以后台附件模式运行，未注册为普通 macOS App

- 优先级：`P0`
- 状态：`待修复`
- 问题：应用当前配置了 `LSUIElement`，启动时还调用 `.accessory` 激活策略，因此无法通过 `Command+Tab` 切换，也不会出现在“强制退出应用程序”窗口中。
- 验收标准：
  - 主窗口打开后，VoiceInput 出现在 `Command+Tab` 应用切换器中。
  - VoiceInput 出现在“强制退出应用程序”窗口中。
  - Dock 中显示正常应用图标，点击后可激活或恢复主窗口。
  - 主窗口关闭后应用可继续保留菜单栏听写能力，不因关闭窗口而退出。
  - 菜单栏常驻、右 Command 听写和非激活 HUD 行为不受影响。
  - 从菜单栏选择“退出”或从系统强制退出后，进程确实结束。
  - `Info.plist` 与运行时 activation policy 不再互相矛盾。

- 验收标准：
  - 当前选中项使用浅色高亮背景、强调色图标和高对比标题。

  - 高亮不使用大面积高饱和颜色，且与内容区强调色一致。
  - 整行点击区域和视觉高亮区域完全一致。

## 6. 笔记

### BUG-NOTE-001 笔记页不是录音实时转录流程

- 优先级：`P0`
- 状态：`待修复`
- 参考图：
  - [空闲态](interaction-references/2026-06-12-feedback/01-notes-idle.png)
  - [录音态](interaction-references/2026-06-12-feedback/02-notes-recording.png)
  - [完成态](interaction-references/2026-06-12-feedback/03-notes-completed.png)
- 问题：当前笔记交互没有体现“点击录音、实时转录、完成并生成记录”的核心流程。
- 验收标准：
  - 空闲态显示“快速记录，随时回看”、主输入区和醒目的录音按钮。
  - 点击录音按钮后立即进入录音状态并开始实时显示转录文本。
  - 录音状态有明确的完成按钮、字符数和录音状态反馈。
  - 点击完成后停止录音，将文本保存为笔记，并在最近记录中立即出现。
  - 完成后编辑区保留可继续编辑的文本。
  - 录音失败、无权限或转录失败时显示明确错误，不静默结束。

  - 无记录时显示居中的图标和“暂无记录”。
  - 有记录时显示标题摘要、日期和时间。
  - 搜索和视图切换按钮位置清晰，点击区域完整。

## 7. 词汇表

### BUG-GLOSSARY-001 选中卡片使用大面积深绿色，视觉过重

- 优先级：`P2`
- 状态：`仍不通过`
- 参考图：[词汇表选中颜色问题](interaction-references/2026-06-12-feedback/11-glossary-selected-color.png)
- 问题：“易错词”选中态使用高饱和深绿色大色块，与全局浅色设置界面不协调。
- 验收标准：
  - 改为浅色背景、细描边或小面积强调色。
  - 标题保持高对比深色文字。
  - 选中态明显，但不使用整块高饱和深色背景。
  - “易错词”和“文本替换”卡片高度、间距和图标容器一致。

### BUG-GLOSSARY-002 易错词暴露内部字段或导入规则不符合一期需求

- 优先级：`P1`
- 状态：`待验证`
- 问题：易错词只应面向用户展示单词或短语，不应展示内部 `general`、优先级、别名等字段。
- 验收标准：
  - 文本框每行输入一个单词或短语。
  - TXT 导入只接受 UTF-8 `.txt`，每行一项。
  - 忽略空行并跳过重复项。
  - 页面不出现 JSON、CSV、分类、优先级和别名入口。

### BUG-GLOSSARY-003 文本替换功能范围不正确

- 优先级：`P1`
- 状态：`待验证`
- 问题：文本替换只需要手动新增和删除，不需要导入导出。
- 验收标准：
  - 保留来源文本和目标文本输入。
  - 支持手动新增和删除。
  - 不显示导入、导出、JSON、CSV 或批量替换入口。
