# VoiceInput 第一期 UI 与交互修复设计

## 目标

修复首页、词汇表、风格和文件转写页面中影响第一期可用性的交互问题，并删除当前不需要的扩展功能。实现应优先保证点击区域明确、操作反馈可辨识、中文文案完整，以及核心流程可直接完成。

## 首页

- 修正侧栏收起按钮在收起状态下的对齐，保证图标始终位于工具栏按钮视觉中心。
- 历史复制后显示醒目的蓝色 Toast，包含复制图标和“已复制”文案。
- 历史删除后显示醒目的红色 Toast，包含删除图标和“已删除”文案。
- 错误反馈仍使用红色警示样式，但与删除 Toast 使用不同图标和文案。
- 点击复制或删除按钮时，不触发历史行的详情选择。

## 词汇表

页面保留两个功能区：

1. **易错词**
   - 词条只向用户展示“单词/短语”，不展示内部 `general` 分类、优先级或别名。
   - 支持在输入框中输入一个或多行词条，点击“添加”后按行保存。
   - 支持选择 `.txt` 文件导入，UTF-8 文本每行一个单词或短语。
   - 忽略空行；同名词条不重复创建。
   - 词条列表支持删除。
2. **文本替换**
   - 保留来源文本、目标文本的手动新增和删除。
   - 不提供导入、JSON、CSV、逐行替换或导出入口。

数据层继续使用现有 `GlossaryTerm` 和 `ReplacementRule`，但第一期 UI 不暴露分类、优先级、匹配阶段等高级字段。新建记录使用稳定的内部默认值。

## 风格

- 左侧每个风格条目的完整矩形区域都可点击，包括文字周围和卡片边缘。
- 点击风格立即：
  1. 设为默认风格；
  2. 加载该风格提示词到编辑区。
- 删除右上角“设为默认”勾选按钮。
- 删除应用风格规则面板及其新增、编辑、删除入口。
- 删除刷新/重置按钮和编辑器下方的模型、参数、调试等非第一期内容。
- 右侧只保留：
  - 当前风格名称；
  - 左侧 Markdown 源码编辑器；
  - 右侧 Markdown 实时预览；
  - 一个“确认”按钮。
- “确认”保存当前风格提示词。提示词按 Markdown 纯文本持久化，预览使用系统 Markdown 解析能力。

## 文件转写

- 状态全部显示中文：
  - `queued`：等待开始
  - `running`：转写中
  - `completed`：已完成
  - `failed`：失败
  - `cancelled`：已取消
- 每个任务根据状态显示明确操作：
  - 播放/暂停：播放或暂停原始音视频文件；
  - 开始：启动等待中的任务；失败或取消后用于重新开始；
  - 复制：结果可用时直接复制转写文本。
- 完成后的主要操作不再使用分享菜单。
- 复制后显示明确 Toast。
- 播放状态由页面级播放器管理，切换任务时停止前一个任务。

## 验证

- ViewModel 单元测试覆盖词条按行导入、TXT 内容解析、重复词处理、状态中文映射和结果复制。
- 样式选择测试覆盖“点击即默认”的行为。
- 运行相关筛选测试及完整 `swift test`。
- 构建并签名 `.build/VoiceInputApp.app`。
- 在真实 App 中逐页检查：首页 Toast 与侧栏收起、词汇表添加/导入/替换、风格点击与 Markdown 预览、文件转写按钮和中文状态。

## 不在本期

- 词条分类、优先级、别名编辑。
- JSON/CSV 词汇表导入导出。
- 文本替换批量导入。
- 应用到风格的自动映射 UI。
- 风格模型、温度、Provider 和调试预览配置。
- 文件转写结果的 Markdown/SRT 分享菜单。
